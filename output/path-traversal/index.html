Path traversal

A security vulnerability that allows attackers to access restricted directories and files on a server.

Description

Path traversal is a type of security vulnerability commonly found in web applications that allows an attacker to read files outside the intended directory. This occurs when an application does not properly sanitize user inputs, enabling an attacker to manipulate file paths. By crafting specific input, such as using '../' sequences, an attacker can traverse the directory structure and access sensitive files that should be out of reach. This can lead to the exposure of confidential information, such as configuration files, user data, or application source code. Path traversal vulnerabilities can be mitigated through proper validation and sanitization of user input, enforcing strict access controls, and employing security mechanisms such as whitelisting acceptable file paths. It's a critical issue that can lead to severe security breaches if not addressed, making it essential for developers to be aware of and protect against such vulnerabilities in their applications.

Examples

  • An attacker sends a request with a path like '../../etc/passwd' to access the server's password file.
  • Exploiting a web application that does not validate file uploads, allowing access to sensitive configuration files.

Additional Information

  • Path traversal vulnerabilities are often listed as part of the OWASP Top Ten security risks.
  • Regular security audits and code reviews can help identify and remediate potential path traversal vulnerabilities.
